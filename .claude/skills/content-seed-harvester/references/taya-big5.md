# TAYA Big 5 — Classification Reference

From Marcus Sheridan's "They Ask, You Answer" framework, adapted for DevOps education content targeting mid-level engineers breaking into DevOps/SRE.

Assign one primary category per pain signal. The goal is a balanced bank — if one category is overrepresented, lean toward underrepresented ones on borderline cases.

---

## 1. Cost/Price

The person is calculating ROI or worried about money.

- Certification costs, bootcamp pricing, salary expectations
- "Is X worth the money?", "How much does Y cost?"
- Cloud spend anxiety, tool licensing confusion
- Career ROI: "Will learning Z actually get me hired?"
- Time investment framing: "Is it worth spending 3 months on..."

**DevOps examples:**
- AWS cert exam fees + practice test costs
- Bootcamp vs self-taught cost comparison
- Expected salary jumps after learning Kubernetes
- Whether a homelab is worth the electricity bill

---

## 2. Problems/Negatives

The person is encountering (or fearing) something going wrong.

- Burnout, on-call horror stories, imposter syndrome
- "This is harder than I expected"
- Tool/process frustrations, broken workflows
- Career fears: "Am I too old?", "Is DevOps dying?"
- Things tutorials don't warn you about
- Feeling stuck, isolated, or unsupported at work

**DevOps examples:**
- On-call rotation destroying work-life balance
- Terraform state corruption nightmares
- Being the only person who cares about CI/CD at their company
- Tutorial hell — completing courses but can't do the job
- Imposter syndrome in first DevOps role

---

## 3. Versus/Comparisons

The person is choosing between options and can't decide.

- Tool A vs Tool B
- Career path A vs B
- Cloud providers, cert paths, learning resources
- "Should I learn X or Y first?"
- Migration decisions: staying vs switching

**DevOps examples:**
- Terraform vs Pulumi vs CloudFormation
- SRE vs DevOps vs Platform Engineering career path
- AWS vs GCP vs Azure for beginners
- CKA vs CKAD — which cert first
- Jenkins vs GitHub Actions vs GitLab CI

---

## 4. Reviews

The person wants an honest, unfiltered assessment.

- Course/bootcamp reviews, cert exam experiences
- Tool evaluations from practitioners (not vendors)
- Employer/role assessments
- "Was the CKA worth it?", "Is Pluralsight still good?"
- Experience reports after switching tools/roles

**DevOps examples:**
- Honest CKA exam experience and study strategy
- Whether KodeKloud/Pluralsight/Udemy courses are good
- What SRE is actually like at a FAANG company
- Whether a DevOps bootcamp led to actual job offers

---

## 5. Best-of Lists

The person wants a curated, ranked set of options.

- "Top tools for...", "Best resources for..."
- "What should every DevOps engineer know?"
- Learning roadmaps, essential skill lists
- "What are the must-know tools in 2026?"
- Starter kits, recommended setups

**DevOps examples:**
- Top 5 CI/CD tools for beginners
- Best programming language for SRE
- Essential Linux commands for DevOps interviews
- Best homelab setup for learning Kubernetes
- Must-read books for aspiring SREs

---

## Tie-Breaking Rules

When a post fits multiple categories:

1. **Problems + Versus** (frustrated while comparing): **Problems wins** — the pain is more actionable than the decision
2. **Cost + Reviews** (evaluating something's value): **Cost wins** — money is the root concern
3. **Reviews + Best-of** (wanting recommendations with opinions): **Reviews wins** — deeper content opportunity
4. **Career direction posts**: Usually **Problems** (fear/anxiety) unless they explicitly list options to compare (then **Versus**)
5. **When still unsure**: Default to **Problems** — it's the largest content gap and what the audience needs most honesty about
6. **Balance rule**: If one category has 3x more entries than the smallest category, prefer the smaller category on borderline cases
